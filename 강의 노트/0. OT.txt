<웹 서버 VS 게임 서버>
웹 서버 : 테이크 아웃 식당 
 ㄴ 실시간 상호작용이 필요없다, 드물게 정보를 요청/갱신 
 ㄴ 식당에서 먼저 손님한테 접근하는 일이 없다
 ㄴ 주문 후 손님이 떠나면 손님의 상태를 잊고 지낸다 (State less)
 ㄴ 웹 서비스 만드는데 사용

게임 서버 : 일반 식당 - 상호작용이 있음
 ㄴ 요청/갱신 횟수가 많다
 ㄴ 실시간 상호작용이 필요하다
 ㄴ 언제라도 직원이 손님한테 접근이 가능 
 ㄴ 손님의 상태가 계속 유지한다 .( State ful)
   ㄴ 어떤 유저가 어떤 행동을 하면 다른 유저들에게도 동기화가 되어야 하고 무엇을 행동하는지 계속 알아야함
     ㄴ 주기적으로 패킷을 받아야함

게임 서버 제작은 게임, 장르에 따라 요구사항이 다르다
 ㄴ 최적의 프레임 워크라는 것이 없다

 ==> 무엇을 고려해야 할까?
 최대 동시 접속자, 장르 및 채널링, 게임 로직, 네트워크, DB, 쓰레드 개수 and 모델, 네트워크 모델 등등
